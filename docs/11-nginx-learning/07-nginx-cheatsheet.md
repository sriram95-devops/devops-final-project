# Guide 07 — NGINX Quick Reference Cheat Sheet

Bookmark this page. Use it while working with NGINX every day.

---

## 1. Core Commands

### On-Prem / Azure VM (Ubuntu)

```bash
# Install
sudo apt install nginx -y

# Start / Stop / Restart / Reload (no downtime)
sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx          # brief downtime
sudo nginx -s reload                  # no downtime — use in production

# Auto-start on boot
sudo systemctl enable nginx

# Test config before applying
sudo nginx -t

# Test and reload in one safe command
sudo nginx -t && sudo nginx -s reload

# Check status
sudo systemctl status nginx

# Check version
nginx -v

# Check what ports NGINX is listening on
sudo ss -tlnp | grep nginx

# View access log live
sudo tail -f /var/log/nginx/access.log

# View error log live
sudo tail -f /var/log/nginx/error.log

# View full effective config (all includes merged)
sudo nginx -T

# Enable a site
sudo ln -s /etc/nginx/sites-available/mysite /etc/nginx/sites-enabled/mysite

# Disable a site
sudo rm /etc/nginx/sites-enabled/mysite
```

### Kubernetes / Ingress

```bash
# List all Ingress objects
kubectl get ingress -A

# List Ingress in a namespace
kubectl get ingress -n dev

# Describe Ingress (see routing rules + events)
kubectl describe ingress <name> -n dev

# View Ingress Controller logs
kubectl logs -n ingress-nginx deployment/ingress-nginx-controller

# View Ingress Controller logs live
kubectl logs -n ingress-nginx deployment/ingress-nginx-controller -f

# Get the actual nginx.conf generated by the controller
kubectl exec -n ingress-nginx deployment/ingress-nginx-controller -- cat /etc/nginx/nginx.conf

# Test routing without DNS (port-forward)
kubectl port-forward -n ingress-nginx svc/ingress-nginx-controller 8080:80
curl -H "Host: shop.local" http://localhost:8080/api/products

# Get External IP of Ingress Controller
kubectl get svc ingress-nginx-controller -n ingress-nginx

# Restart Ingress Controller
kubectl rollout restart deployment/ingress-nginx-controller -n ingress-nginx

# Scale Ingress Controller
kubectl scale deployment ingress-nginx-controller -n ingress-nginx --replicas=3
```

## 2. Config Snippets

### Minimal reverse proxy

```nginx
server {
    listen 80;
    server_name _;
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### Route multiple services by path

```nginx
server {
    listen 80;
    server_name _;

    location /api/products { proxy_pass http://127.0.0.1:3001; }
    location /api/users    { proxy_pass http://127.0.0.1:3002; }
    location /             { proxy_pass http://127.0.0.1:3000; }
}
```

### Load balancing (round-robin across 3 servers)

```nginx
upstream myapp {
    server 10.0.0.1:3000;
    server 10.0.0.2:3000;
    server 10.0.0.3:3000;
}
server {
    listen 80;
    location / { proxy_pass http://myapp; }
}
```

### Load balancing (sticky sessions — same client → same server)

```nginx
upstream myapp {
    ip_hash;                   # ← client IP always goes to same server
    server 10.0.0.1:3000;
    server 10.0.0.2:3000;
}
```

### Load balancing (weighted — send more traffic to a stronger server)

```nginx
upstream myapp {
    server 10.0.0.1:3000 weight=3;   # ← gets 3x more traffic
    server 10.0.0.2:3000 weight=1;
}
```

### HTTPS with self-signed cert

```nginx
server {
    listen 80;
    return 301 https://$host$request_uri;
}
server {
    listen 443 ssl;
    ssl_certificate     /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header X-Forwarded-Proto https;
    }
}
```

### Rate limiting (max 10 requests/second per IP)

```nginx
http {
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;  # define zone in http block

    server {
        location /api/ {
            limit_req zone=api_limit burst=20 nodelay;   # apply in location block
        }
    }
}
```

### Block specific IPs

```nginx
location / {
    deny  192.168.1.100;      # block this IP
    allow all;                # allow everyone else
    proxy_pass http://backend;
}
```

### Allow only specific IPs (internal admin page)

```nginx
location /admin {
    allow 10.0.0.0/8;         # allow internal network
    deny all;                 # block everyone else
    proxy_pass http://admin-backend;
}
```

### Custom error pages

```nginx
server {
    error_page 502 /502.json;
    error_page 504 /504.json;

    location = /502.json {
        internal;
        return 502 '{"error":"service_unavailable","code":502}';
        add_header Content-Type application/json;
    }
    location = /504.json {
        internal;
        return 504 '{"error":"gateway_timeout","code":504}';
        add_header Content-Type application/json;
    }
}
```

### NGINX status page (for monitoring)

```nginx
location /nginx_status {
    stub_status;
    allow 127.0.0.1;
    deny all;
}
```

### Serve static files with caching headers

```nginx
location /static/ {
    root /var/www/myapp;
    expires 30d;                                      # cache for 30 days
    add_header Cache-Control "public, immutable";
}
```

### Large file upload support

```nginx
server {
    client_max_body_size 100M;   # allow up to 100MB request body
}
```

### WebSocket support

```nginx
location /ws {
    proxy_pass http://websocket-backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;      # ← required for WebSocket
    proxy_set_header Connection "Upgrade";       # ← required for WebSocket
    proxy_set_header Host $host;
}
```

## 3. Kubernetes Ingress Snippets

### Minimal path-based Ingress

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp-ingress
  namespace: dev
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
  - host: myapp.local
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api-service
            port:
              number: 80
```

### Ingress with TLS

```yaml
spec:
  tls:
  - hosts:
    - myapp.yourdomain.com
    secretName: myapp-tls
  rules:
  - host: myapp.yourdomain.com
    http:
      paths: ...
```

### Common Ingress Annotations

| Annotation | Effect |
|-----------|--------|
| `nginx.ingress.kubernetes.io/rewrite-target: /` | Strip path prefix before forwarding |
| `nginx.ingress.kubernetes.io/use-regex: "true"` | Allow regex in path rules |
| `nginx.ingress.kubernetes.io/proxy-body-size: "50m"` | Allow request bodies up to 50MB |
| `nginx.ingress.kubernetes.io/proxy-read-timeout: "120"` | Backend read timeout in seconds |
| `nginx.ingress.kubernetes.io/proxy-send-timeout: "120"` | Backend send timeout in seconds |
| `nginx.ingress.kubernetes.io/rate-limit-rps: "10"` | 10 requests/second rate limit |
| `nginx.ingress.kubernetes.io/ssl-redirect: "false"` | Disable HTTP→HTTPS redirect |
| `nginx.ingress.kubernetes.io/backend-protocol: "HTTPS"` | Backend uses HTTPS |
| `nginx.ingress.kubernetes.io/affinity: "cookie"` | Sticky sessions via cookie |
| `cert-manager.io/cluster-issuer: letsencrypt-prod` | Auto-issue TLS cert via cert-manager |

## 4. File Locations Reference

| Environment | nginx.conf | Sites config | Logs |
|-------------|-----------|--------------|------|
| Ubuntu (on-prem/VM) | `/etc/nginx/nginx.conf` | `/etc/nginx/sites-available/` | `/var/log/nginx/` |
| Windows (manual install) | `C:\nginx\conf\nginx.conf` | `C:\nginx\conf\` | `C:\nginx\logs\` |
| Docker container | `/etc/nginx/nginx.conf` | `/etc/nginx/conf.d/` | stdout |
| Kubernetes pod | `/etc/nginx/nginx.conf` (inside controller pod) | Generated from Ingress objects | `kubectl logs` |

## 5. Generate a Self-Signed Certificate

```bash
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/server.key \
  -out    /etc/nginx/ssl/server.crt \
  -subj "/CN=localhost/O=MyOrg/C=US"
```

## 6. Install Helm Chart Reference

### Minikube (addon)

```bash
minikube addons enable ingress
```

### Minikube / AKS (Helm)

```bash
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update
helm install ingress-nginx ingress-nginx/ingress-nginx \
  --namespace ingress-nginx \
  --create-namespace \
  --set controller.replicaCount=2 \
  --wait
```

```bash
helm uninstall ingress-nginx -n ingress-nginx
kubectl delete namespace ingress-nginx
```

## 7. Error Code Quick Reference

| Code | Name | Meaning | First Place to Check |
|------|------|---------|---------------------|
| 200 | OK | Success | — |
| 301 | Moved Permanently | HTTP→HTTPS redirect | Intentional |
| 400 | Bad Request | Client sent malformed request | Client-side issue |
| 403 | Forbidden | Access blocked | File permissions, IP deny rules |
| 404 | Not Found | No matching location block | NGINX config, Ingress paths |
| 413 | Too Large | Request body exceeds limit | Add `client_max_body_size` |
| 499 | Client Closed | Client disconnected | Client timeout, load balancer timeout |
| 502 | Bad Gateway | Backend not running | Check backend process/pod |
| 503 | Service Unavailable | No healthy backends | Check all pods running |
| 504 | Gateway Timeout | Backend too slow | Increase `proxy_read_timeout` |
